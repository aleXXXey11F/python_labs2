### Краткое описание классов

**Bus** — базовый класс, представляющий автобус на маршруте. Содержит общие атрибуты (номер маршрута, вместимость, средняя скорость, имя водителя, текущее количество пассажиров, статус на маршруте) и методы (посадка/высадка, начало/конец маршрута, расчёт времени в пути, рейтинг эффективности, расчёт стоимости проезда).

**CityBus** — городской автобус (наследник Bus). Добавляет тип маршрута (обычный/экспресс) и наличие валидатора/кондуктора. Предоставляет метод получения расписания.

**TouristBus** — туристический автобус (наследник Bus). Добавляет наличие Wi-Fi, туалета и имя экскурсовода. Предоставляет метод определения уровня комфорта.

**SchoolBus** — школьный автобус (наследник Bus). Добавляет наличие сопровождающего и название школы. Предоставляет метод рейтинга безопасности.

**Основные возможности:**
- **Наследование** — дочерние классы расширяют функциональность базового без дублирования кода (используют `super()`)
- **Полиморфизм** — метод `calculate_fare()` работает по-разному в зависимости от типа автобуса
- **Переопределение методов** — `__str__()`, `get_info()`, `board_passenger()` (в SchoolBus)
- **Проверка типов** — `isinstance()` для фильтрации коллекции
- **Интеграция с коллекцией** — объекты всех типов хранятся в `Fleet` из ЛР-2

### Методы классов

#### Методы базового класса Bus:
| Метод | Описание |
|-------|----------|
| `board_passenger()` | Посадка одного пассажира |
| `alight_passenger()` | Высадка одного пассажира |
| `start_route()` | Отправление на маршрут |
| `end_route()` | Завершение маршрута |
| `calculate_travel_time(distance)` | Расчёт времени в пути |
| `get_efficiency_rating()` | Рейтинг эффективности (низкая/средняя/высокая загрузка) |
| `calculate_fare(distance)` | Расчёт стоимости проезда (полиморфный) |
| `get_info()` | Получение словаря с информацией (полиморфный) |

#### Новые атрибуты и методы дочерних классов:
| Класс | Новые атрибуты | Новые методы |
|-------|----------------|--------------|
| `CityBus` | `route_type`, `has_validator` | `get_route_schedule()` |
| `TouristBus` | `has_wifi`, `has_toilet`, `tour_guide_name` | `get_comfort_level()` |
| `SchoolBus` | `has_escort`, `school_name` | `get_safety_rating()` |

#### Переопределённые методы (полиморфизм):
| Класс | `calculate_fare(distance)` | `__str__` (дополнительная информация) |
|-------|----------------------------|----------------------------------------|
| `Bus` | 20 + 10 × distance | базовая (маршрут, водитель, пассажиры, статус) |
| `CityBus` | 30 руб (или 24 со скидкой) | + тип маршрута, наличие валидатора |
| `TouristBus` | (100 + 20×distance) × коэффициент комфорта | + удобства (WiFi/туалет), экскурсовод, уровень комфорта |
| `SchoolBus` | 0 (бесплатно) | + наличие сопровождающего, школа, рейтинг безопасности |

### Выводы

В ходе выполнения лабораторной работы были освоены:

1. **Механизм наследования** — создана иерархия из одного базового и трёх дочерних классов.
2. **Использование `super()`** — вызов конструктора базового класса без дублирования кода.
3. **Полиморфизм** — метод `calculate_fare()` реализован по-разному в каждом классе, вызывается через единый интерфейс.
4. **Переопределение методов** — `__str__()`, `get_info()`, `board_passenger()`.
5. **Проверка типов** — `isinstance()` для фильтрации коллекции.
6. **Интеграция с коллекцией из ЛР-2** — объекты всех типов корректно хранятся в `Fleet`, поддерживаются все операции.

**Результат:** Разработана гибкая и расширяемая иерархия классов, демонстрирующая основные принципы ООП: наследование, инкапсуляцию и полиморфизм.
