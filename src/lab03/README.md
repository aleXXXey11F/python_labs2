# Лабораторная работа №3
## Наследование и иерархия классов (Python)

### Выбранная предметная область
**Транспорт**

### Иерархия классов
Bus (базовый класс)
├── CityBus (городской автобус)
└── TouristBus (туристический автобус)

### Реализованные классы

#### 1. Базовый класс `Bus` (из ЛР-1, расширен)
- **Атрибуты**: номер маршрута, вместимость, средняя скорость, имя водителя, количество пассажиров, статус на маршруте.
- **Методы**: посадка/высадка, начало/окончание маршрута, расчёт времени в пути, рейтинг эффективности.

#### 2. Производный класс `CityBus` (городской автобус)
**Новые атрибуты:**
- `number_of_stops` – количество остановок на маршруте
- `has_air_conditioning` – наличие кондиционера

**Новый метод:**
- `calculate_stop_density()` – плотность остановок (на 10 км)

**Переопределённый метод:**
- `calculate_travel_time(distance)` – добавляет время на остановки (0.5 мин на остановку)

#### 3. Производный класс `TouristBus` (туристический автобус)
**Новые атрибуты:**
- `has_toilet` – наличие туалета
- `luggage_capacity` – вместимость багажа (кг)

**Новый метод:**
- `calculate_comfort_level()` – уровень комфорта (0–10)

**Переопределённый метод:**
- `calculate_travel_time(distance)` – увеличивает время на 10% для комфортного стиля вождения

### Интеграция с коллекцией из ЛР-2
Класс `Fleet` (автопарк) успешно хранит объекты всех трёх типов (`Bus`, `CityBus`, `TouristBus`).  
Добавлены методы фильтрации по типу с помощью `isinstance()`.

### Полиморфное поведение
- Метод `calculate_travel_time()` ведёт себя по-разному в каждом классе, вызывается единообразно.
- Метод `get_efficiency_rating()` (унаследованный) работает одинаково для всех.
- Нет проверок `type()` – используется полиморфизм через переопределение.

### Демонстрация работы (`demo.py`)

#### Сценарий 1 – Создание объектов иерархии
Показано создание базового автобуса, городского и туристического. Вызваны новые методы дочерних классов.

*(Скриншот 1)*

#### Сценарий 2 – Полиморфизм (расчёт времени в пути)
Для одинаковой дистанции и базовой скорости каждый тип автобуса выдаёт своё время. Городской – дольше из-за остановок, туристический – из-за надбавки на комфорт.

*(Скриншот 2)*

#### Сценарий 3 – Коллекция Fleet с разными типами
В один автопарк добавлены объекты всех трёх классов. Вызов `calculate_travel_time()` для каждого элемента коллекции демонстрирует полиморфизм.

*(Скриншот 3)*

#### Сценарий 4 – Фильтрация по типу
С помощью `isinstance()` из общей коллекции выделены только городские автобусы, только туристические и только чистые базовые.

*(Скриншот 4)*

#### Сценарий 5 – Единый полиморфный интерфейс
Вызваны методы `get_efficiency_rating()` (единое поведение) и `calculate_travel_time()` (разное поведение) для всех объектов без использования условных операторов.

*(Скриншот 5)*

### Выводы
- Реализовано наследование: два производных класса добавляют свои атрибуты и методы.
- Использован `super()` для вызова конструктора базового класса.
- Переопределены методы для достижения полиморфизма.
- Коллекция из ЛР-2 без изменений работает с любыми наследниками `Bus`.
- Применена фильтрация по типу и проверка `isinstance()`.
- Достигнуто полиморфное поведение без анти-паттерна `if type == ...`.
