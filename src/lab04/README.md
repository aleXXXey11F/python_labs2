# Лабораторная работа №4
## Интерфейсы и абстрактные классы (Python)

### Выбранная предметная область
**Транспорт**

В систему автобусов, построенную в ЛР1–ЛР3, добавлены интерфейсы поведения (`Printable`, `Comparable`). Классы автобусов и билетов реализуют один или оба интерфейса, демонстрируя полиморфизм через контракты. Коллекция `ExtendedFleet` умеет фильтровать, выводить и сортировать объекты исключительно через интерфейсные методы.

### Реализованные интерфейсы

- **Printable(ABC)** – требует метод `to_string() -> str`
- **Comparable(ABC)** – требует метод `compare_to(other: Comparable) -> int`

### Реализация в классах

| Класс                     | Реализуемые интерфейсы | Особенности `to_string()` / `compare_to()` |
|---------------------------|------------------------|---------------------------------------------|
| `PrintableCityBus`        | `Printable`            | Выводит номер, загрузку, низкопольность, кондиционер |
| `ComparableIntercityBus`  | `Comparable`           | Сравнение по средней скорости |
| `AdvancedElectricBus`     | `Printable`, `Comparable` | Выводит ёмкость батареи, запас хода; сравнение по `battery_capacity` |
| `Ticket`                  | `Printable`, `Comparable` | Выводит ID, маршрут, цену и статус; сравнение по цене |

Коллекция `ExtendedFleet` (наследует `Fleet`):
- `get_printable()` → список `Printable`
- `get_comparable()` → список `Comparable`
- `print_all_printable()` – полиморфный вызов `to_string` для всех
- `sort_comparable()` – сортировка с использованием `compare_to`
